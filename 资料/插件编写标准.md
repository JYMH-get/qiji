# 📖 漫剧 (Manju) 插件开发 API 标准 v4.1

本文档为漫剧插件开发的官方 API 参考手册。所有插件必须遵循此规范以确保与系统核心调度器的完美兼容。

---

## 一、 开发顺序规约 (Execution Order)

插件代码应严格遵循以下物理组织顺序，严禁随意调整：

1.  **引用与前置声明**：
    引入依赖（`fetch`）、声明 `PLUGIN_NAME`、定义基础常量（如 `DEFAULT_API_BASE`、`REGISTER_URL`）。
2.  **文件元信息**：
    导出 `id`, `name`, `version`, `type` 等基础属性。
3.  **模型映射 (Models)**：
    定义模型列表、颜色、比例、时长、素材限制（`mediaConstraints`）及引用模板（`mentionTemplate`）。
4.  **配置表单 (Schema)**：
    定义用户设置界面项（API Key, Base URL, 调试开关, 轮询设置等）。
5.  **核心业务逻辑**：
    实现 `createTask`（提交）、`queryTask`（轮询）。
6.  **标准辅助函数**：
    实现 `truncateForLog`、`logDebug` 以及素材索引处理逻辑。

---

## 二、 插件属性规范 (Plugin Properties)

插件必须通过 `module.exports` 导出的基础元数据：

| 属性名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | `string` | 是 | 插件唯一标识符，建议使用下划线命名（如 `jianmeng_pro`） |
| `name` | `string` | 是 | 在客户端界面显示的插件名称 |
| `version` | `string` | 是 | 语义化版本号，如 `1.0.0` |
| `type` | `string` | 是 | 枚举值：`image` (图像), `video` (视频), `audio` (音频) |
| `supportImageHost` | `boolean` | 否 | 默认 `true`。若为 `true`，`params.images` 为公网 URL；为 `false` 则为 Base64 |

---

## 三、 模型规格配置 (Models Configuration)

`models` 数组中的对象属性详细定义：

| 属性名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | `string` | 传递给后端 API 的真实模型 ID |
| `name` | `string` | 用户在 UI 界面看到的模型名称 |
| `color` | `string` | 界面显示的颜色类名（如 `text-blue-400 font-bold`） |
| `ratios` | `string[]` | 支持的比例，如 `['1:1', '16:9', '9:16', 'auto']` |
| `durations` | `number[]` | (仅视频) 支持的时长选项（秒），如 `[5, 10, 15]` |
| `qualities` | `string[]` | (仅图像) 支持的质量选项，如 `['HD', '4K']` |
| `maxImages` | `number` | 该模型允许上传的素材总数上限 |
| `mediaConstraints` | `object` | **素材拦截约束**。定义 `image`, `video`, `audio` 的限制属性 |
| `mentionTemplate` | `string` | 提示词引用模板，如 `@{rawType}_file_{index}` |

---

## 四、 配置表单选项 (ConfigSchema)

用于动态生成插件设置界面的字段属性：

| 字段名 | 可选值 / 类型 | 说明 |
| :--- | :--- | :--- |
| `type` | `text`, `password`, `number`, `boolean`, `button` | 界面组件类型 |
| `key` | `string` | 存储在 `config` 对象中的属性名 |
| `label` | `string` | 界面显示的中文标签名称 |
| `defaultValue` | `any` | 初始默认值 |
| `placeholder` | `string` | 输入框内的占位提示文本 |
| `min` / `max` | `number` | (仅数字类型) 允许输入的数值范围 |
| `action` | `string` | (仅按钮类型) 点击时执行的插件方法名 |

> [!NOTE]
> **内置保留键 (Reserved Keys)**：
> * `apiKey`：存放插件 API 秘钥。
> * `apiBase`：存放 API 代理地址。
> * `useImageHost`：是否启用图床（类型 `boolean`，默认 `true`），若为 `false` 则强制采用 Base64 数据直传。
> * `debugMode`：是否开启控制台调试日志输出。

---

## 五、 核心方法 API 接口 (Method Specs)

### 1. `createTask(config, params)`
**输入参数 `params` 对象字段：**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `prompt` | `string` | 包含占位符的原始提示词 |
| `model` | `string` | 用户选择的模型真实 ID |
| `ratio` | `string` | 用户选择的画幅比例（如 `16:9`） |
| `duration` | `number` | (视频) 生成时长 |
| `images` | `array` | 素材数组。成员为 `{ url: string, type: string }` |
| `rendererTaskId`| `string` | 系统任务 ID，用于进度上报 |
| `generate_audio`| `boolean`| 是否生成视频音频（默认 `true`） |

### 2. `queryTask(config, taskId, params)`
**返回值对象定义：**

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `status` | `string` | 状态枚举：`success`, `failed`, `processing`, `pending` |
| `progress` | `number` | 0 - 100 的整数进度百分比 |
| `video_url` | `string` | (视频) 生成成功后的结果视频链接 |
| `image_url` | `string` | (图像) 生成成功后的结果图片链接 |
| `error` | `string` | 失败时的详细错误描述信息 |
| `pollInterval` | `number` | (可选) 建议下一次查询的秒数（默认 15） |
| `pollMaxAttempts`| `number` | (可选) 最大允许查询的次数（建议 60-80） |

## 六、 生产级开发案例 (Case Study)

```javascript
/**
 * production_standard_plugin.js
 * 当前模板仅做参考，实际请求体以官方接口为准
 */
const PLUGIN_NAME = '[Manju-Pro]';
const DEFAULT_API_BASE = 'https://api.jian1.vip';
const REGISTER_URL = 'https://api.jian1.vip/register?aff=EZ0G';

module.exports = {
    // 2. 元数据
    id: "pro_video_v4",
    name: "生产级视频标准插件",
    version: "4.1.0",
    type: "video",
    supportImageHost: true,

    // 3. 模型定义
    models: [
        {
            id: 'pro-model-v2',
            name: '满血生产模型',
            color: 'text-indigo-400 font-bold',
            ratios: ['auto', '16:9', '9:16', '1:1', '21:9'],
            durations: [5, 10, 15],
            maxImages: 12,
            mediaConstraints: {
                image: { max: 9, label: '图片', maxSize: 15 * 1024 * 1024 },
                video: { max: 1, label: '视频', maxSize: 30 * 1024 * 1024 }
            },
            mentionTemplate: "@{rawType}_file_{index}"
        }
    ],

    // 4. 配置表单
    configSchema: [
        { key: "apiKey", label: "API Key", type: "password", placeholder: "sk-..." },
        ...(REGISTER_URL ? [{ key: "registerUrl", label: "获取 API 令牌", type: "button", url: REGISTER_URL }] : []),
        { key: "apiBase", label: "代理地址", type: "text", defaultValue: DEFAULT_API_BASE },
        { key: "useImageHost", label: "是否启用图床", type: "boolean", defaultValue: true },
        { key: "debugMode", label: "调试模式", type: "boolean", defaultValue: false },
        // 轮询配置 (这两个是成对可选出现的。如果不使用自定义轮询参数，可以不写)
         { key: "pollInterval", label: "轮询间隔(秒)", type: "number", defaultValue: 15, min: 5 },
         { key: "pollMaxAttempts", label: "最大轮询次数", type: "number", defaultValue: 60, min: 1 }
    ],

    // 5. 核心方法
    async createTask(config, params) {
        const apiKey = (config.apiKey || "").trim();
        const apiBase = (config.apiBase || DEFAULT_API_BASE).replace(/\/$/, "");
        
        // 安全校验与进度上报
        if (/[^\x00-\x7F]/.test(apiKey)) throw new Error("密钥包含非法字符");
        if (config.reportProgress) config.reportProgress(params.rendererTaskId, "正在准备并提交任务...");

        const requestBody = {
            model: params.model,
            prompt: params.prompt,
            ratio: params.ratio || 'auto',
            duration: params.duration || 5,
            generate_audio: params.generate_audio !== false
        };

        // 自动化多模态素材构造
        (params.images || []).forEach((asset, idx) => {
            const type = asset.type || 'image';
            requestBody[`${type}_file_${idx + 1}`] = asset.url;
        });

        const res = await fetch(`${apiBase}/v1/videos`, {
            method: 'POST',
            headers: { 'Authorization': `Bearer ${apiKey}`, 'Content-Type': 'application/json' },
            body: JSON.stringify(requestBody)
        });

        const data = await res.json();
        if (!res.ok) throw new Error(data.error?.message || "任务提交失败");
        return data.task_id || data.id;
    },

    async queryTask(config, taskId, params = {}) {
        const apiBase = (config.apiBase || DEFAULT_API_BASE).replace(/\/$/, "");
        const res = await fetch(`${apiBase}/v1/videos/${taskId}`, {
            headers: { 'Authorization': `Bearer ${config.apiKey}` }
        });

        // 核心网关高可用容错
        if ([502, 503, 504, 521].includes(res.status)) return { status: 'pending', pollInterval: 15 };
        
        const data = await res.json();
        const videoUrl = data.url || data.video_url || data.result?.video_url || (data.urls && data.urls[0]);
        
        let status = 'processing';
        if (['success', 'succeeded', 'completed', 3].includes(data.status)) status = 'success';
        else if (['failed', 'error', 4].includes(data.status)) status = 'failed';

        // 结果提前判定逻辑
        if (videoUrl && status !== 'failed') status = 'success';

        return {
            status,
            progress: data.progress || (status === 'success' ? 100 : 50),
            video_url: videoUrl,
            error: data.error?.message || data.message,
            pollInterval: 15
        };
    }

};
